# Luminir Document Parser - 주요 작업

## 1. UI/UX 디자인 및 구현

- 사내 문서 변환 도구 디자인
- 데스크톱 웹 환경 최적화 (최소 1280px)
- Tailwind CSS 기반 모던 UI
- SVG 기반 아이콘 시스템 (lucide-react)
- 반응형 레이아웃
- 사이드바 네비게이션 구조

## 2. 파일 업로드 및 변환 시뮬레이션

- 드래그 앤 드롭 업로드
- 멀티 파일 업로드 (HWP/PDF/이미지)
- 실시간 진행률 표시
- 페이지 단위 변환 진행 모니터링
- 파일별 상태 관리 (대기/변환 중/완료/실패)
- 전체 진행률 표시
- 업로드된 파일 목록 UI
- 파일 타입별 아이콘 표시

## 3. 파일 상세 결과 페이지

- HTML 표 렌더링
- Markdown 선형화 가독성 향상
- 이미지 설명 스타일링
- 원본 텍스트 표시
- 페이지 단위 결과 네비게이션
- 탭 기반 결과 분류 (표/선형화/이미지/원본)
- 다운로드 기능
- 삭제 기능
- 뒤로가기 네비게이션

## 4. RAG 기반 질의응답 시스템

- 채팅 인터페이스
- 질문/답변 히스토리
- 참조 문서 자동 표시
- 페이지 번호 기반 참조
- 관련도 점수 표시
- 로딩 상태 애니메이션
- 대화 컨텍스트 유지
- 응답 시간 표시

## 5. 파일 관리 페이지

- 전체 파일 목록 조회
- 검색 기능 (파일명)
- 필터링 (상태별, 파일 타입별)
- 날짜 범위 선택
- 페이지네이션
- 파일 상태 배지 표시
- 파일 상세 보기
- 파일 삭제 기능
- 정렬 기능

## 6. Dashboard UI 개발

### 전체 작업 현황 대시보드

- 전체 작업 수 통계 카드
- 완료된 파일 수
- 진행 중인 파일 수
- 대기 중인 파일 수
- 실패한 파일 수
- Daily 처리 파일 수 (Trend Chart)
- 평균 처리 시간 표시
- 성공률 통계

### 실시간 모니터링 위젯

- 현재 진행 중인 작업 수
- 시스템 리소스 사용률 (CPU, 메모리)
- 큐 대기 작업 수
- 실시간 업데이트 상태

### 작업 통계 차트

- 일별 처리량 차트 (Area Chart)
- 파일 유형별 처리 통계 (Bar Chart)
- 모델별 성능 비교
- 시간대별 처리 통계

### 최근 작업 내역

- 최근 처리된 파일 목록 (상위 5개)
- 파일명, 유형, 페이지 수
- 처리 시간, 사용 모델
- 상태 배지 표시
- 날짜/시간 정보
- 필터링 옵션 (상태별, 파일 타입별)

### 에러 로그 모니터링

- 에러 유형별 통계 그래프 (VLM 타임아웃, 메모리 부족, 변환 실패, 파일 형식 오류)
- 최근 발생한 에러 내역 (상위 5개)
- 전체 에러 로그 페이지
- 에러 상세 정보 모달 (스택 트레이스, 파일 정보 및 발생 시각, 권장 조치 안내)
- 검색 기능 (파일명)
- 필터링 (에러 유형별)
- 페이지네이션

## 7. 설정 페이지

- **VLM 모델 설정**: 모델 선택 (gpt-5.2, gpt-5-mini, deepseek-ocr-2), 타임아웃 설정, 재시도 횟수 설정
- **변환 옵션**: 이미지 품질 설정, 페이지 범위 지정, 출력 형식 선택
- **시스템 설정**: 동시 처리 수 제한, 메모리 임계값 설정, 로그 레벨 설정

## 8. 공통 컴포넌트 시스템

- **StatusBadge**: 파일 상태 시각화 (완료/변환 중/실패/대기 중), 일관된 색상 및 아이콘 시스템
- **FileTypeIcon & FileTypeBadge**: 파일 타입별 아이콘 (PDF/HWP/이미지), 파일 타입 배지 표시
- **ErrorTypeBadge**: 에러 유형별 배지, 색상 코드 시스템
- **ErrorDetailModal**: 에러 상세 정보 모달, 스택 트레이스 표시, 파일 정보 및 발생 시각, 권장 조치 안내, 재시도 기능
- **StatCard**: 통계 카드 컴포넌트, 아이콘 및 수치 표시, 서브타이틀 지원

## 9. 주요 기능 플로우

### 파일 변환 플로우

```
파일 업로드 (드래그 앤 드롭 또는 선택)
→ 대기 큐에 추가
→ 페이지 단위 변환 진행
→ 실시간 진행률 표시
→ 변환 완료 후 결과 확인
→ RAG 기반 질의응답 가능
```

### 모니터링 플로우

```
대시보드에서 전체 현황 확인
→ 실시간 통계 및 차트 조회
→ 에러 발생 시 알림
→ 에러 로그 상세 확인
→ 권장 조치에 따른 재처리
```
